Dossier Jury - Phase 2

ChainCacao : Notes de Démonstration

Ce document présente les acteurs simulés et le flux démontré dans le prototype de la plateforme de traçabilité Premium ChainCacao.

1. Acteurs Simulés

La démonstration met en scène quatre rôles distincts pour illustrer la chaîne de valeur complète :

  • Agriculteur (Farmer) : Acteur sur le terrain. Il enregistre la récolte (poids, variété, coordonnées GPS) depuis son mobile, souvent en zone blanche (hors-ligne).
  • Coopérative : Acteur intermédiaire. Gère le volume des lots reçus, effectue le premier contrôle de conformité, valide les lots et les transfère.
  • Vérificateur / Auditeur : Acteur de contrôle Premium. Il certifie ou rejette les lots après un audit de traçabilité, déclenchant l'ancrage Blockchain.
  • Consommateur final (Public) : Acteur de bout de chaîne. Scanne le code du lot pour visualiser l'historique transparent et luxueux de son produit.

2. Flux Démontré (Workflow Complet)

Étape 1 : Enregistrement sur le terrain (Agriculteur)

L'agriculteur se connecte et crée un nouveau lot de cacao. La démonstration illustre la capacité Offline-First : la création est instantanée, le brouillon est sauvegardé localement, et la synchronisation avec le serveur (batch processing) s'effectue automatiquement dès le retour de la connexion réseau.

Étape 2 : Validation en Coopérative (Kanban)

Connexion avec le compte Coopérative. Nous montrons le tableau de bord interactif (Kanban) où les lots nouvellement reçus apparaissent dans la colonne "À valider".
L'action de validation est effectuée par un simple glisser-déposer (Drag & Drop) ou via un bouton. Nous mettons en évidence l'Optimistic UI : l'interface se met à jour instantanément sans temps de chargement, offrant une expérience fluide et "Premium".

Étape 3 : Certification (Vérificateur)

L'auditeur se connecte et accède à tous les lots validés par les coopératives. Grâce au correctif de pagination massive, aucun lot n'est masqué. Le vérificateur glisse le lot vers la colonne "Certifiés". Cette action génère une preuve de traçabilité immuable.

Étape 4 : Consultation Publique (Consommateur)

Sans aucune connexion requise, nous saisissons le code unique d'un lot certifié sur la page publique (/public/verify).
Nous démontrons l'affichage de la Timeline animée (réalisée avec Framer Motion) qui retrace avec élégance et transparence le parcours du cacao, répondant aux exigences d'une image de marque de luxe.

3. Points Techniques Clés Soulevés

  • Architecture Hors-Ligne (Offline-First) : Utilisation de IndexedDB et d'un contexte de synchronisation personnalisé (SyncContext) pour garantir le fonctionnement sans internet.
  • Optimistic UI : Les actions de changement d'état (validation, certification) s'affichent instantanément pour l'utilisateur, l'appel API (et la gestion du cache) s'exécutant en arrière-plan.
  • Design Système Premium : Implémentation d'un thème clair (Cocoa/Cream/Gold) sur-mesure avec variables CSS, effets Glassmorphism, et animations pour véhiculer l'aspect "Luxe" exigé par la marque.
