ChainCacao

Notes de démonstration du prototype

Support de prise de parole pour la vidéo ou la soutenance. Ce document résume le scénario de démo et les points à montrer.

1. Message d’ouverture

Présenter ChainCacao comme une interface métier pour tracer les lots de cacao, sécuriser les opérations et permettre une vérification publique sans compte.

Insister sur les 4 idées clés: capture terrain, offline-first, workflow par rôle, et consultation publique.

2. Démonstration rapide

  • Se connecter avec un rôle agriculteur, coopérative ou vérificateur.
  • Créer un lot avec photo, poids, date et coordonnées.
  • Montrer la sauvegarde locale du brouillon et la file de synchronisation.
  • Ouvrir la liste des lots puis le détail d’un lot.
  • Basculer vers la vérification publique et saisir un code lot.

3. Ce qu’il faut souligner au jury

  • Le projet fonctionne en mode mobile-first et reste lisible hors ligne.
  • Les routes sont protégées selon le rôle de l’utilisateur.
  • Les mutations peuvent être rejouées quand la connexion revient.
  • Le style reste cohérent et accessible, avec de gros points d’action.
  • Le README doit expliquer la solution, les technologies et l’exécution locale.

4. Rappel de soumission

Avant dépôt final, vérifier que le dossier Drive principal contient bien le sous-dossier PHASE_2_DEMIFINAL, les liens du prototype, du README, la vidéo de démonstration et le document de notes exporté en PDF.
